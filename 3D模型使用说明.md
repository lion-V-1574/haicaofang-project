## 📁 文件放置

### 第一步：确认文件位置

将你的两个文件放到 `images` 文件夹中：

```
你的项目/
├── images/
│   ├── 草房子结构图1.gltf  ← 主文件（必须）
│   ├── 草房子结构图1.bin   ← 数据文件（必须）
│   └── ...其他图片
└── index2.html
```

**⚠️ 重要：**
- 两个文件必须在同一个文件夹
- 文件名必须完全一致（包括中文字符）
- GLTF 文件会自动引用 BIN 文件

## 🔧 自定义配置

### 修改模型路径

如果你的文件名不同，修改这一行：

```javascript
const modelPath = 'images/草房子结构图1.gltf';
```

改成你的文件名：

```javascript
const modelPath = 'images/你的文件名.gltf';
```

### 修改模型大小

找到这一行：

```javascript
const targetSize = 4;  // 目标大小
```

改成：

```javascript
const targetSize = 6;  // 更大
// 或
const targetSize = 2;  // 更小
```

### 修改旋转速度

找到这一行：

```javascript
controls.autoRotateSpeed = 2;  // 旋转速度
```

改成：

```javascript
controls.autoRotateSpeed = 5;  // 更快
// 或
controls.autoRotateSpeed = 1;  // 更慢
```

### 修改初始视角

找到这一行：

```javascript
camera.position.set(5, 5, 5);
```

改成：

```javascript
camera.position.set(8, 6, 8);  // 更远
// 或
camera.position.set(3, 3, 3);  // 更近
```

## ❌ 常见问题

### 问题 1：模型不显示

**症状：** 一直显示"正在加载 3D 模型..."

**排查步骤：**
1. 按 F12 打开控制台
2. 查看是否有红色错误
3. 检查文件路径是否正确
4. 确认使用 HTTP 服务器访问

**可能原因：**
- 文件路径错误
- 文件名不匹配
- 没有使用 HTTP 服务器
- BIN 文件缺失

### 问题 2：显示"模型加载失败"

**症状：** 显示红色错误提示

**解决方法：**
1. 检查文件是否存在
2. 检查文件名是否正确（包括中文字符）
3. 确认 GLTF 和 BIN 文件都在 images 文件夹
4. 查看控制台的详细错误信息

### 问题 3：模型太大或太小

**解决方法：**
修改 `targetSize` 参数：

```javascript
const targetSize = 4;  // 调整这个数字
```

### 问题 4：模型位置不对

**解决方法：**
点击"重置"按钮，或刷新页面

### 问题 5：控制按钮不工作

**解决方法：**
1. 等待模型完全加载
2. 刷新页面重试
3. 查看控制台是否有 JavaScript 错误

## 📊 性能优化

### 如果模型加载慢

1. **压缩模型文件**
   - 使用 Blender 重新导出
   - 降低模型精度
   - 减少顶点数量

2. **优化贴图**
   - 如果有贴图，压缩图片
   - 降低贴图分辨率

3. **调整渲染质量**

```javascript
// 降低像素比
renderer.setPixelRatio(1);  // 从 2 改为 1

// 关闭阴影
renderer.shadowMap.enabled = false;
```
